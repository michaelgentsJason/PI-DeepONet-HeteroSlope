# AGENT.md

## 1. 任务定义

本项目以现有 `DeepONet-WaterFlow-Soils-main` 为代码基座，目标是将其从：

- 均质土体
- 矩形二维入渗域
- 给定表面通量的前向 PI-DeepONet

改造为：

- 非均质边坡
- 降雨入渗场景
- 不依赖外部数值模拟器生成训练标签
- 以物理约束训练为主的 PI-DeepONet / branch-trunk operator model

当前阶段不直接追求“最终论文级全部结果”，而是先建立一个可持续迭代的研究代码框架，使后续模型、损失函数、几何表示、采样策略和实验脚本都能稳定扩展。

## 2. 已确认的事实

### 2.1 现有代码库能力

当前仓库核心代码位于 `DeepONet_Codes/`，主要特征如下：

- 框架：JAX
- 网络：branch-trunk DeepONet 风格 PI-DeepONet
- 物理方程：二维 Richards equation 残差
- 输入分支：表面通量时间序列
- trunk 输入：`(t, x, z)`
- 土体参数：通过 `Richards.py` 中固定 `soil_type` 给定，为均质参数
- 边界与几何：默认单位矩形域，顶部通量边界、左右无流、底部自由排水近似
- 训练数据：主要是 collocation / IC / BC 点，不依赖大规模标签

### 2.2 参考论文对本任务的启发

`case/Water Resources Research - 2025 - Gong - Physics Informed Neural Network With Enhanced Parameter State Coupling for Inverse.pdf`

已抽取并确认的关键点：

- 论文核心关注“非均质参数场”和“不确定边界条件”共同存在时的 PINN 建模。
- 其关键结论不是“仅仅加入参数输入就够了”，而是“参数特征必须被显式耦合到状态预测过程中”。
- 论文中更有效的做法是 branch-trunk 结构下的增强耦合，即参数特征不能只通过 PDE loss 间接影响状态。
- 论文虽然主要是 1D 异质土柱与反演问题，但其“显式参数-状态耦合”的思想适合迁移到本任务。

## 4. 研究问题重新表述

### 4.1 目标问题

给定：

- 边坡几何
- 降雨过程（时间序列）
- 非均质土体参数场表示
- 初始条件
- 边界条件

训练一个 physics-informed DeepONet，使其在任意时空点 `(t, x, z)` 上预测：

- 压力头 `h` 或 matric potential
- 可选地派生体积含水率 `theta`
- 可选地派生 Darcy 通量 `q`

### 4.2 第一阶段的简化建模假设

为了保持问题“简单但正确”，第一阶段采用以下假设：

1. 流动为二维变饱和渗流，不耦合力学，不做稳定性分析。
2. 土体各向同性。
3. 暂不考虑蒸发、根系吸水、裂隙流、滞回。
4. 优先仅让 `K_s(x,z)` 空间变化，`alpha, n, theta_r, theta_s` 先保持全局常数。
5. 降雨边界以 Neumann flux 形式给定。
6. 不依赖 FEM / HYDRUS / FreeFEM 生成训练标签。
7. 允许后续使用少量参考解做 sanity check，但不进入主训练闭环。

这组假设直接对应参考论文中“优先异质 `K_s`、其他参数先固定”的策略，也符合老师要求的低复杂度路线。

## 5. 从现有仓库到新课题的关键改造

### 5.1 现有仓库不满足新课题的地方

当前代码的不足点：

- `Richards.py` 只支持均质土参数，无法表达空间异质性。
- 训练数据生成函数默认矩形域采样，不能表达边坡几何。
- 上边界只是一条水平边界，未区分坡面边界与其它边界。
- branch 输入只编码表面通量，不编码土体异质性。
- 物理残差默认采用均质参数函数，无法表达 `K(h; x, z)`。

### 5.2 新课题的最小可行架构

建议采用“增强耦合的异质 PI-DeepONet”，最小可行版本如下：

- `branch_rain`：编码降雨历时曲线
- `branch_hetero`：编码非均质参数场
- `trunk`：编码 `(t, x, z)` 以及必要的几何特征
- `fusion`：显式耦合，而不是简单拼接

推荐两种可执行实现路径：

1. 简化版
   - 将“降雨特征 + 异质参数特征”拼成单一 branch 输入
   - trunk 仍接收 `(t, x, z)`
   - 最终通过点积或 gated product 输出

2. 稍强版
   - 降雨和异质性分开编码
   - 再投影到同维 latent space
   - 与 trunk latent 做显式融合

默认先做第 1 条，原因是：

- 改动小
- 与现有代码兼容性最好
- 已能体现“异质参数显式进入 forward prediction”

## 6. 非均质边坡的表示方案

### 6.1 几何表示

第一阶段不直接上复杂不规则网格，而采用“单位参考域 + 几何映射”的方案：

- 在规范化域中定义边坡截面
- 用掩码或边界判定函数区分：
  - 坡面边界
  - 坡顶平台
  - 侧边界
  - 底边界

推荐最小实现：

- 用一个梯形或三角形坡体截面
- 通过解析表达式判断点是否在域内
- 通过参数化边界采样函数生成各类 BC 点

这样可以避免引入网格剖分器，同时保留“边坡”几何特征。

### 6.2 非均质参数场表示

优先级从简到强：

1. 分区常数场
   - 例如坡体分为 2 到 4 个土层或土区
   - 最容易实现，适合建立首个工作版本

2. 低维随机场
   - 采用 KLE 或 RBF 基函数表示 `log(K_s(x,z))`
   - 用少量系数控制连续空间异质性
   - 更贴近参考论文思想

执行决策：

- 第一版先支持“分区非均质”
- 第二版再引入 KLE 随机场

原因：

- 边坡几何本身已经增加了复杂度
- 若同时一次性引入 2D KLE、复杂坡面边界和多参数反演，任务会明显超纲

## 7. 物理方程与损失函数设计

### 7.1 状态变量

优先预测压力头 `h(t,x,z)`，再通过 van Genuchten-Mualem 关系得到：

- `theta(h; params)`
- `K(h; params)`

### 7.2 异质情况下的参数处理

当 `K_s = K_s(x,z)` 时：

- `K(h, x, z)` 不再只是 `h` 的函数
- PDE 残差需要正确处理空间异质性项

因此实现上必须区分：

- constitutive relation 中对 `h` 的导数
- 参数场本身对空间坐标的变化

第一阶段如果使用“分区常数场”，则：

- 子区内部可视为常参数
- 区界面附近可先避免直接采样，降低梯度奇异问题

这是最实用的起步方式。

### 7.3 损失项

基础损失建议包含：

- `L_res`：Richards PDE collocation residual
- `L_ic`：初始条件
- `L_rain_bc`：坡面降雨通量边界
- `L_noflux_bc`：非入渗边界无流条件
- `L_outlet_bc`：若定义坡脚出逸/排水边界，则加对应约束

可选损失：

- `L_obs`：若后续加入少量伪观测或现场观测
- `L_interface`：若采用分区非均质，可在界面两侧加入弱连续性约束
- `L_mass`：整体质量守恒或累计入渗约束

## 8. 数据与采样策略

### 8.1 不使用数值模拟标签的训练方式

训练集不由数值解构成，而是由“场景参数 + collocation / BC / IC 点”构成。

每个场景至少包含：

- 一条降雨历时曲线
- 一组异质性参数
- 固定边坡几何参数
- 一套 IC / BC / PDE 采样点

### 8.2 场景采样

建议训练分布：

- 降雨强度：多段常数或分段线性 hyetograph
- 降雨持续时间：随机采样
- 非均质参数：先从少量土区组合开始
- 初始负压头：在一个合理区间内随机

### 8.3 采样细节

- 域内 collocation 点动态重采样
- 坡面边界点按弧长均匀采样
- 其它边界分开采样，避免一个函数混装全部边界逻辑
- 对尖角附近单独控制采样密度，避免训练初期不稳定

## 12. 风险与控制

### 12.1 主要风险

- 2D Richards PINN 本身训练困难
- 边坡几何增加边界采样复杂度
- 非均质参数场会显著增加优化难度
- 若一次引入过多自由度，模型容易不收敛

### 12.2 控制策略

- 先做分区异质，不直接上高维随机场
- 先做固定边坡几何，不同时学习几何
- 先让 forward problem 成立，不先做 inverse problem
- 优先保证训练闭环可运行，再谈精度提升
